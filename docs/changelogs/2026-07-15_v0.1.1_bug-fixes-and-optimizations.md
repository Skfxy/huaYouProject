# v0.1.1 项目问题检查与 Bug 修复

- **日期**: 2026-07-15

## 问题描述
使用 webapp-testing 技能对项目进行全面检查，发现多个影响游戏正常运行的 Bug 和可以优化的点，需要修复以确保游戏可正常游玩。

## 🔴 严重问题（必须修复）

### 1. 游戏无法正常启动 - 核心流程 Bug
- [x] **问题位置**: `packages/web/src/views/HomeView.vue` 和 `packages/web/src/views/GameView.vue`
- **问题描述**:
  - HomeView 中点击"开始游戏"时先调用 `gameStore.startGame()`，然后路由跳转到 /game
  - GameView 的 onMounted 中有条件判断 `if (!gameStore.game.isStarted)`，但此时 isStarted 已经是 true
  - 导致 `resetEngine()` 和 `startGame()` 没有被调用，`currentEvent` 保持为 null
  - 结果：直接显示"第一章 · 完"的结束界面，无法进行游戏
- **修复方案**: 统一游戏初始化逻辑，移除 HomeView 中的 `gameStore.startGame()`，让 GameView 完全负责初始化，或修改条件判断确保引擎始终正确启动

### 2. 序章结束后无事件可触发
- [x] **问题位置**: `packages/web/src/data/events/chapter1.json` 和 `packages/web/src/data/events/daily-events.json`
- **问题描述**:
  - `scene_1_5`（序章完结事件）的选项只有 `timeAdvance: 1`，没有 `nextEvent`
  - 序章结束时是第1天早晨，推进时段后变成下午
  - 但所有日常事件（daily_morning、daily_afternoon 等）都设置了 `minDay: 2`
  - 日期事件（date_*）也有 `minDay: 2`
  - 结果：序章结束后 selectNextEvent() 返回 null，又会显示结束界面
- **修复方案**:
  - 将日常事件的 `minDay` 改为 1，或
  - 为序章结束后的第1天下午/晚上添加专门的过渡事件，或
  - 让 scene_1_5 的选项直接跳到第2天早晨

### 3. 存档加载后无法恢复当前剧情事件
- [x] **问题位置**: `packages/web/src/stores/save.store.ts`
- **问题描述**:
  - 存档时保存了 `currentEvent` ID（字符串）
  - 加载存档时只设置了 `gameStore.game.currentEvent = data.currentEvent`
  - 但事件引擎的 `currentEvent` ref 没有被更新，也没有从 allEvents 中查找并设置对应的事件对象
  - 结果：读档后也会显示结束界面或没有剧情
- **修复方案**: 在 loadSaveData 后需要调用事件引擎的方法来根据事件ID恢复当前事件，或者在 GameView 的 onMounted 中增加存档加载后的事件恢复逻辑

## 🟡 中等问题（建议修复）

### 4. 时间信息重复显示
- [x] **问题位置**: `packages/web/src/components/game/StatusBar.vue` 和 `packages/web/src/views/GameView.vue`
- **问题描述**: 状态栏组件 StatusBar 内部已经显示了"第X天、时段、星期"，但 GameView 中又重复显示了一套日期信息
- **修复方案**: 移除 GameView 中重复的 date-info 区域，或让 StatusBar 不显示时间信息，保持单一数据源

### 5. 存档页返回按钮逻辑不合理
- [x] **问题位置**: `packages/web/src/views/SaveView.vue`
- **问题描述**: 在游戏中点击"💾 存档"进入存档页后，点击"← 返回"按钮会直接返回首页（/），而不是返回游戏
- **修复方案**: 使用 `router.back()` 或根据来源判断返回路径，如果是从游戏进入则返回 /game

### 6. 设置页面功能未真正实现
- [x] **问题位置**: `packages/web/src/views/SettingsView.vue`
- **问题描述**:
  - 文字速度、音乐音量、音效音量只是本地 ref 状态
  - 没有持久化到 localStorage
  - 没有与 DialogBox 的打字速度、音频系统关联
  - 音乐/音效功能尚未实现
- **修复方案**: 创建专门的 settings store，将设置持久化，并与 DialogBox 等组件关联

### 7. 开场事件时段与实际时间不匹配
- [x] **问题位置**: `packages/web/src/stores/game.store.ts` 和 `packages/web/src/data/events/chapter1.json`
- **问题描述**:
  - 初始 period 是 "morning"
  - scene_0_1 描述的是"加班的深夜"
  - 开场剧情是深夜场景，但游戏时间初始化为早晨，时间逻辑不一致
- **修复方案**: 开场时将时间设为 night，或调整开场剧情的时间描述，或在序章中正确处理时间推进（从深夜睡觉→第二天早晨醒来）

### 8. TypeScript 类型忽略（@ts-ignore）
- [x] **问题位置**: `packages/web/src/views/HomeView.vue`
- **问题描述**: 使用了 `// @ts-ignore` 来忽略 homeBg 图片导入的类型问题
- **修复方案**: 添加正确的类型声明文件（如 vite-env.d.ts 或在 types/index.d.ts 中声明图片模块），而不是使用 @ts-ignore

### 9. 事件选项缺少条件过滤
- [x] **问题位置**: `packages/web/src/components/game/OptionList.vue` 和 `packages/shared/src/types/game.types.ts`
- **问题描述**: EventOption 类型定义中有 `condition?: EventCondition` 字段，但 OptionList 组件没有根据条件过滤或禁用不符合条件的选项
- **修复方案**: 在 OptionList 中检查每个 option 的 condition，不满足条件的选项应该被隐藏或禁用

## 🟢 优化建议（提升体验）

### 10. 代码格式问题
- [x] **问题描述**: 有大量 prettier 换行符警告（CRLF vs LF 问题）
- **修复方案**: 运行 `pnpm format` 自动修复

### 11. 使用自定义弹窗替代 alert/confirm
- [x] **问题位置**: `packages/web/src/views/SaveView.vue`、`packages/web/src/views/SettingsView.vue`
- **问题描述**: 当前使用 `alert()` 和 `confirm()` 原生弹窗，体验较生硬
- **修复方案**: 使用项目已有的 `packages/web/src/components/common/BaseModal.vue` 组件来实现自定义弹窗

### 12. 确认加载状态/加载页元素
- [x] **问题描述**: 虽然 main.ts 中有 app-loading 的处理，需要确认 index.html 中是否有对应的加载动画元素

### 13. 自动存档提示位置冲突
- [x] **问题位置**: `packages/web/src/views/GameView.vue`
- **问题描述**: 有三个固定定位的提示（effect-hint、special-hint、autosave-hint），分别在 top: 80px、120px、160px，当多个提示同时出现时可能重叠或位置混乱
- **修复方案**: 统一管理提示显示，使用队列或堆叠布局

### 14. 游戏流程中缺少返回主菜单的二次确认
- [x] **问题位置**: `packages/web/src/views/GameView.vue`
- **问题描述**: 在游戏中点击"← 返回"直接返回首页，可能丢失进度（虽然有自动存档）
- **修复方案**: 添加确认弹窗

### 15. 技能 maxExperience 初始化问题
- [x] **问题位置**: `packages/web/src/stores/player.store.ts`
- **问题描述**: 初始化时 `maxExperience` 硬编码为 `config.experienceRequired[1] || 100`
- **修复方案**: 应使用 `getSkillUpgradeExperience` 函数来正确计算

### 16. 首屏背景图片加载优化
- [x] **问题描述**: 首页背景图 home-bg.jpg 可能较大
- **修复方案**: 考虑添加加载占位或使用更优化的图片格式（webp）
