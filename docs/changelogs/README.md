# 更新日志 Changelogs

本目录记录项目每次迭代、优化、Bug 修复、功能新增的详细内容。

## 使用规范

### 文件命名格式
```
YYYY-MM-DD_vX.Y.Z_short-description.md
```
- `YYYY-MM-DD`：日期
- `vX.Y.Z`：版本号（遵循语义化版本）
- `short-description`：简短英文描述，用连字符分隔

示例：`2026-07-15_v0.1.1_bug-fixes-and-optimizations.md`

### 日志类型标签
- 🔧 **Bug修复** - 修复现有问题
- ✨ **功能优化** - 改进现有功能
- 🎨 **UI改进** - 界面/样式调整
- ⚡ **性能优化** - 性能提升
- 🆕 **新功能** - 新增功能
- 📝 **文档** - 文档更新
- 🔨 **重构** - 代码重构

### 状态标记
所有任务项使用 checkbox 标记完成状态：
- `- [ ]` 未开始
- `- [ ]` 进行中
- `- [x]` 已完成

### 文档模板
每次新建日志时，请使用以下模板：

```markdown
# 更新标题

- **日期**: YYYY-MM-DD
- **版本**: vX.Y.Z
- **类型**: 标签
- **状态**: - [ ] 进行中 / - [x] 已完成

## 问题/需求描述
<描述本次更新要解决的问题或需求>

## 更新内容
- [ ] 具体修改项 1
- [ ] 具体修改项 2
- [ ] 具体修改项 3

## 涉及文件
- `path/to/file1.ts`
- `path/to/file2.vue`

## 备注
<其他补充说明>
```
