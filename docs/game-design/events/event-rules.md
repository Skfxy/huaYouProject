# 事件触发规则

> 本章定义事件触发的核心规则，适用于所有章节

---

## 触发机制概述

### 事件触发流程

```
┌─────────────────────────────────────────────────────────┐
│                    事件触发主循环                         │
└─────────────────────────────────────────────────────────┘
                           │

                           │
              ┌────────────┴────────────┐
              │                         │
              ▼                         ▼
       ┌────────────┐            ┌────────────┐
       │  时间条件不满足│           │  时间条件满足 │
       └────────────┘            └────────────┘
              │                         │
              ▼                         ▼
       ┌────────────┐            ┌────────────┐
       │  跳过该事件  │            │  检查地点条件 │
       └────────────┘            └────────────┘
                                      │
                             ┌────────┴────────┐
                             │                 │
                             ▼                 ▼
                      ┌────────────┐    ┌────────────┐
                      │  地点条件不满足│   │  地点条件满足 │
                      └────────────┘    └────────────┘

                      ┌────────────┐    ┌────────────┐
```

## 条件类型

### 时间条件

**时段定义**：
| 时段 | 时间范围 | 说明 |
|------|----------|------|
| 早晨 | 6:00-12:00 | 精力充沛，适合学习 |
| 下午 | 12:00-18:00 | 正常效率 |
| 晚上 | 18:00-24:00 | 精力下降，适合社交 |
| 深夜 | 0:00-6:00 | 精力耗尽，避免熬夜 |

**星期条件**：
- 周一至周五：工作日，触发工作/学习事件
- 周六周日：休息日，触发社交/娱乐事件

**日期条件**：
- 特定日期触发特殊事件（如发薪日、节日）
- 日期范围触发持续事件（如面试季）

### 地点条件

**地点类型**：
| 地点 | 可触发事件 |
|------|----------|
| 宿舍 | 社交、休息、娱乐 |
| 图书馆 | 学习、偶遇 |
| 食堂 | 社交、日常 |
| 公司 | 工作、职场社交 |
| 面试地点 | 求职相关 |

**地点切换**：
- 玩家可主动切换地点
- 特定事件会强制切换地点
- 地点切换消耗时间

### 状态条件

**数值条件**：
```
精力 > X：需要精力充足
心情 > X：需要心情良好
健康 > X：需要身体健康
存款 > X：需要经济宽裕
```

**技能条件**：
```
编程等级 >= X：需要编程技能达标
运营等级 >= X：需要运营技能达标
社交能力 >= X：需要社交能力达标
```

**道具条件**：
```
拥有道具A：需要获得特定道具
未拥有道具B：需要没有特定道具
```

**剧情条件**：
```
已选择路线A：需要选择特定主线
已触发事件X：需要触发过特定事件
好感度 >= X：需要NPC好感度达标
```

## 概率计算

### 基础概率

| 事件类型 | 基础概率 | 说明 |
|----------|----------|------|
| 主线事件 | 100% | 必须触发，推动剧情 |
| 高优先级 | 50% | 高概率触发 |
| 普通优先级 | 30% | 正常触发概率 |
| 低优先级 | 15% | 低概率触发 |
| 隐藏事件 | 10% | 需要特定条件 |

### 概率修正

**状态修正**：
```
心情修正 = 心情 > 70 ? +20% : (心情 < 30 ? -20% : 0%)
精力修正 = 精力 > 80 ? +10% : (精力 < 30 ? -10% : 0%)
```

**技能修正**：
```
社交修正 = 社交能力 * 2%
```

**道具修正**：
```
拥有特定道具：+10%~+30%
```

**时间修正**：
```
连续学习2小时后：学习事件概率-30%
熬夜时：负面事件概率+30%
```

### 最终概率计算

```
最终概率 = 基础概率 × (1 + 心情修正) × (1 + 精力修正) × (1 + 技能修正) × (1 + 道具修正) × (1 + 时间修正)
```

**概率上限**：最终概率不超过100%
**概率下限**：最终概率不低于5%

## 冷却机制

### 事件冷却

| 事件类型 | 冷却时间 | 说明 |
|----------|----------|------|
| 主线事件 | 无 | 随剧情触发 |
| 高优先级 | 1天 | 同一事件每天最多触发1次 |
| 普通优先级 | 2天 | 同一事件每2天最多触发1次 |
| 低优先级 | 3天 | 同一事件每3天最多触发1次 |
| 隐藏事件 | 无 | 可重复触发 |

### 类型冷却

| 事件类型 | 冷却时间 | 说明 |
|----------|----------|------|
| 正面事件 | 1天 | 同一天最多触发2个正面事件 |
| 负面事件 | 1天 | 同一天最多触发2个负面事件 |
| 社交事件 | 2天 | 同一NPC每2天最多触发1次社交事件 |

### 冷却重置

- 每天0点重置每日冷却
- 每周一重置每周冷却
- 特殊事件可重置特定冷却

## 优先级系统

### 事件优先级

| 优先级 | 权重 | 说明 |
|--------|------|------|
| 必须触发 | 100 | 主线剧情事件，必须触发 |
| 高 | 60 | 重要事件，高概率触发 |
| 普通 | 30 | 正常事件，正常触发 |
| 低 | 15 | 次要事件，低概率触发 |
| 隐藏 | 10 | 特殊事件，需特定条件 |

### 同时触发处理

当多个事件同时满足触发条件时：

1. **优先级最高者优先**：必须触发 > 高 > 普通 > 低 > 隐藏
2. **同优先级随机选择**：权重相同时随机选择
3. **每日触发上限**：每天最多触发3个事件

### 事件覆盖

- 高优先级事件可覆盖低优先级事件
- 主线事件可覆盖所有其他事件
- 玩家可选择跳过非主线事件

## 互斥机制

### 事件互斥

**同一事件互斥**：
- 同一事件不会连续触发
- 同一事件冷却时间内不会触发

**类型互斥**：
- 正面事件和负面事件不会同时触发
- 社交事件和独处事件不会同时触发

**剧情互斥**：
- 选择A后，与A互斥的事件B不会触发
- 特定选择会关闭某些事件路径

### 互斥示例

```
事件1（开黑） ↔ 事件5（面试准备）
- 选择开黑后，面试准备事件不会触发
- 选择面试准备后，开黑事件不会触发

事件3（余额） → 事件4（emo）
- 余额事件可能触发emo事件
- 但emo事件不会触发余额事件
```

## 特殊触发规则

### 连锁触发

某些事件会触发后续事件：

```
事件A触发 → 事件B触发条件达成 → 事件B触发
```

**示例**：
- 获得学长联系方式 → 学长内推事件触发条件达成
- 通过面试 → 入职准备事件触发

### 条件触发

某些事件需要特定条件才能触发：

```
事件X触发条件：
1. 已完成事件A
2. 技能等级 >= 2
3. 好感度 >= 50
4. 时间 >= 第10天
```

### 强制触发

主线事件强制触发，不受概率和冷却限制：

```
主线事件触发流程：
1. 到达特定时间点
2. 显示事件
3. 玩家必须做出选择
4. 推进剧情
```

## 调参建议

### 概率调整

- 正面事件概率略高于负面事件，保持游戏乐趣
- 重要节点（如面试前）增加正面事件概率
- 低谷期（如心情<30）减少负面事件概率

### 冷却调整

- 高优先级事件冷却时间可缩短，增加触发频率
- 低优先级事件冷却时间可延长，避免频繁触发
- 特殊事件可无冷却，增加趣味性

### 优先级调整

- 主线事件优先级必须最高，确保剧情推进
- 重要事件优先级应高于普通事件
- 隐藏事件优先级可适当提高，增加发现感

### 互斥调整

- 互斥事件应明确提示玩家选择影响
- 互斥关系应符合逻辑，避免不合理
- 可设置部分互斥，允许有限的多事件触发
