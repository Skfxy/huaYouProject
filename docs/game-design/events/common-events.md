## 详细事件设计

### 事件 common_001：早起打卡

**触发条件**：
```
时间条件：早晨6-7点起床
状态条件：无
概率：满足条件时，30%概率触发
```

**事件描述**：
**【场景】**
你比平时早起了一会儿，看着窗外的阳光，感觉今天会是美好的一天。

**【内心独白】**
"早起的感觉真好，今天要加油！"

**选项设计**：
- 早起打卡 → 心情+3，健康+1

**后续影响**：
- 增加「早起打卡」成就进度
- 连续早起7天解锁「自律达人」成就

## 事件触发规则

### 触发概率计算

```
最终概率 = 基础概率 × 条件修正 × 随机因子
```

**条件修正**：
- 心情高：正面事件概率+20%，负面事件概率-20%
- 心情低：正面事件概率-20%，负面事件概率+20%
- 精力充沛：所有事件概率+10%
- 精力不足：所有事件概率-10%

**随机因子**：
- 0.8-1.2之间的随机数

### 事件冷却机制

| 事件类型 | 冷却时间 | 说明 |
|----------|----------|------|
| 正面事件 | 1天 | 同一类正面事件每天最多触发1次 |
| 负面事件 | 2天 | 同一类负面事件每2天最多触发1次 |
| 中性事件 | 无 | 可重复触发 |
| 特殊事件 | 3天 | 重要事件冷却时间更长 |

### 事件互斥机制

- 正面事件和负面事件不会同时触发
- 同一类型事件不会连续触发
- 特殊事件优先级最高，会覆盖普通事件

### 事件优先级

| 优先级 | 权重 | 说明 |
|--------|------|------|
| 必须触发 | 1.0 | 触发条件达成后100%触发 |
| 高 | 0.6 | 高概率触发 |
| 普通 | 0.3 | 正常触发概率 |
| 低 | 0.1 | 低概率触发 |

## 事件数值平衡

### 正面事件总效果

- 每周预期触发2-3个正面事件
- 每个正面事件心情+3~+10
- 每周心情预期增长+10~+25

### 负面事件总效果

- 每周预期触发1-2个负面事件
- 每个负面事件心情-3~-10
- 每周心情预期下降-5~-15

### 净效果

- 如果玩家表现好（学习、社交），心情净增长+5~+10/周
- 如果玩家表现差（熬夜、孤立），心情净下降-5~-10/周

### 平衡建议

- 正面事件略多于负面事件，保持游戏乐趣
- 重要节点（如面试前）增加正面事件概率
- 低谷期（如心情<30）减少负面事件概率
