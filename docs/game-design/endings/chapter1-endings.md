# 第一章结局分支

> 本章定义第一章的三个结局，以及触发条件

---

## 结局总览

| 结局ID | 结局名称 | 结局类型 | 触发条件 |
|--------|----------|----------|----------|
| ending_001 | 踌躇满志 | 好结局 | 转行路线 + 心情 > 60 |

## 结局A：踌躇满志

### 基本信息

| 属性 | 内容 |
|------|------|
| 结局ID | ending_001 |
| 结局名称 | 踌躇满志 |
| 结局类型 | 好结局 |
| 解锁章节 | 第二章 · 面试季 |

### 触发条件

```
主条件：
1. 选择转行程序员路线（选项A）
2. 选择运营产品路线（选项C）

附加条件（满足任一）：
1. 心情 > 60
2. 编程/运营等级 >= 1
3. 学长好感度 >= 50

排除条件：
1. 心情 <= 30（触发坏结局）
2. 健康 <= 40（影响结局质量）
```

### 结局描述

**【场景】**
毕业典礼结束，你站在校门口，看着来来往往的学弟学妹。

**【旁白】**
你收拾好行李，带着全新的目标踏出校门。2018年的夏天很热，但你心里更热。

**【内心独白】**
"这一次，我不会再错过了。"
"互联网的红利还在，我要抓住时代的尾巴。"
"三年后，我一定要让那个未来的自己后悔！"

**【旁白】**
你深吸一口气，迈出了改变人生的第一步。身后的校园渐渐远去，前方的道路虽然未知，但你从未如此确信——这就是你要走的路。

### 结局效果

| 属性 | 变化 |
|------|------|
| 心情 | +20（充满希望） |
| 精力 | +10（动力十足） |
| 技能 | 编程/运营等级保留 |
| 道具 | 获得「决心」道具 |

### 解锁内容

- 第二章 · 面试季
- 程序员路线剧情线
- 运营产品路线剧情线
- 新的随机事件池

### 结局评价

**S级结局**：
- 心情 > 80
- 技能等级 >= 2
- 学长好感度 >= 70
- 解锁隐藏剧情

**A级结局**：
- 心情 > 60
- 技能等级 >= 1
- 解锁标准剧情

**B级结局**：
- 心情 > 50
- 技能等级 >= 0
- 解锁基础剧情

### 基本信息

| 属性 | 内容 |
|------|------|
| 结局ID | ending_002 |
| 结局名称 | 稳中求进 |
| 结局类型 | 中结局 |
| 解锁章节 | 第二章 · 双面人生 |

### 触发条件

```
主条件：
1. 选择边工作边学路线（选项B）

2. 存款 > 3000（生活更稳定）
3. 编程等级 >= 0（有学习基础）

排除条件：

2. 健康 <= 30（影响结局质量）
```

### 结局描述

**【场景】**
你按时去工厂报到，成为了一名普通的机械工程师。

**【旁白】**
白天，你在工厂里拧螺丝、画图纸、开会。晚上，你回到出租屋，打开电脑，开始敲代码。

**【内心独白】**
"日子很苦，但每多学一天，我就离目标更近一步。"
"总有一天，我会跳出这个圈子。"
"这次，我不会再放弃了。"

**【旁白】**
你看着屏幕上的代码，虽然还很简陋，但你知道，这就是你改变命运的钥匙。窗外的夜色很深，但你的心里有一团火在燃烧。

### 结局效果

| 属性 | 变化 |
|------|------|
| 心情 | +10（平稳） |
| 精力 | -5（有些疲惫） |
| 存款 | +4500（第一个月工资） |
| 技能 | 编程等级保留 |

### 解锁内容

- 第二章 · 双面人生
- 稳扎稳打路线剧情线
- 工作+学习的平衡机制
- 新的随机事件池

### 结局评价

**S级结局**：
- 心情 > 70
- 存款 > 4000
- 技能等级 >= 1
- 解锁隐藏剧情

**A级结局**：
- 心情 > 50
- 存款 > 3000
- 解锁标准剧情

**B级结局**：
- 心情 > 40
- 存款 > 2000
- 解锁基础剧情

## 结局C：浑浑噩噩

### 基本信息

| 属性 | 内容 |
|------|------|
| 结局ID | ending_003 |
| 结局名称 | 浑浑噩噩 |
| 结局类型 | 坏结局 |
| 解锁内容 | 游戏结束，可重新开始 |

### 触发条件

```
主条件（满足任一）：
1. 选择享受路线（选项D）
2. 心情持续低迷（连续7天心情 < 30）
3. 健康 <= 20（生病导致）

附加条件（加剧坏结局）：
1. 技能等级 = 0（没有学习）
2. 存款 < 1000（经济困难）
3. 社交能力 = 0（孤立自己）
```

### 结局描述

**【场景】**
毕业典礼结束，你站在校门口，看着来来往往的学弟学妹。

**【旁白】**
你又像当年一样，稀里糊涂毕了业，稀里糊涂上了班。好像什么都没变，又好像有什么不一样了。

**【内心独白】**
"算了，就这样吧……"
"也许我本来就不是那块料。"
"那个关于'重来一次'的梦，我渐渐记不清了……"

**【旁白】**
你低头看了看手机，屏幕上的日期是2018年6月20日。你知道，这一天，你又一次错过了改变命运的机会。意识开始模糊，你好像又听到了那阵电流的滋滋声……

### 结局效果

| 属性 | 变化 |
|------|------|
| 心情 | -20（绝望） |
| 精力 | -10（疲惫） |
| 健康 | -5（亚健康） |
| 技能 | 全部清零 |

### 解锁内容

- 游戏结束画面
- 重新开始选项
- 结局回顾功能
- 隐藏成就「轮回者」

### 结局评价

**C级结局**：
- 选择享受路线
- 心情 > 40
- 解锁基础结局画面

**D级结局**：
- 心情持续低迷
- 心情 < 30
- 解锁悲剧结局画面

**E级结局**：
- 健康 <= 20
- 触发「轮回」结局
- 解锁隐藏剧情提示

## 结局触发检测

### 检测时机

- 第20天结束时
- 玩家选择结束第一章时
- 特殊事件触发时（如健康归零）

### 检测顺序

1. **检查排除条件**：健康 <= 20 → 强制坏结局
2. **检查主条件**：根据选择判断结局类型
3. **检查附加条件**：确定结局质量等级
4. **计算结局评分**：综合各项指标

### 结局评分公式

```
结局评分 = 心情权重×0.3 + 技能权重×0.3 + 存款权重×0.2 + 社交权重×0.2

心情权重 = 心情 / 100
技能权重 = (编程等级 + 运营等级) / 10
存款权重 = 存款 / 10000
社交权重 = 社交能力 / 10
```

### 结局等级划分

| 评分范围 | 结局等级 | 说明 |
|----------|----------|------|
| 0.8-1.0 | S级 | 完美结局 |
| 0.6-0.8 | A级 | 优秀结局 |
| 0.4-0.6 | B级 | 良好结局 |
| 0.2-0.4 | C级 | 普通结局 |
| 0.0-0.2 | D级 | 悲剧结局 |

## 调参建议

### 结局触发条件调整

- 好结局条件不宜太苛刻，让玩家有成就感
- 坏结局条件要合理，让玩家感到遗憾但不沮丧
- 中结局要作为默认路径，让大部分玩家能到达

### 结局质量调整

- S级结局要有特殊奖励，激励玩家追求完美
- A级结局要作为标准体验，让玩家满意
- B级结局要作为及格线，让玩家不失望
- C/D级结局要作为教训，让玩家想重玩

### 结局与后续关联

- 好结局要为后续章节提供更多可能性
- 坏结局要让玩家有重玩的动力
- 结局选择要影响第二章的起点和难度
