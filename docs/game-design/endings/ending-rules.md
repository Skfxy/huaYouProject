# 结局判定逻辑

> 本章定义结局判定的核心规则，适用于所有章节

---

## 结局系统概述

### 结局类型

| 类型 | 说明 | 数量 |
|------|------|------|
| 好结局 | 成功达成目标，获得奖励 | 每章1-2个 |
| 中结局 | 基本达成目标，获得基础奖励 | 每章2-3个 |
| 坏结局 | 未达成目标，需要重玩 | 每章1-2个 |
| 隐藏结局 | 特殊条件触发，获得特殊奖励 | 每章1个 |

### 结局触发时机

- 章节结束时
- 特殊事件触发时
- 玩家主动选择时
- 数值达到极端时（如健康归零）

## 强制结局触发

### 健康归零

**触发条件**：
```
健康 <= 0
```

**结局类型**：坏结局

**结局效果**：
- 强制休息3天
- 心情-20
- 精力-10
- 触发「生病」事件

**特殊处理**：
- 如果在章节末尾触发，直接进入坏结局
- 如果在章节中间触发，强制休息后继续游戏

### 心情崩溃

**触发条件**：
```
心情 <= 0
```

**结局类型**：坏结局

**结局效果**：
- 触发「崩溃」事件
- 心情锁定为0
- 所有事件触发概率-50%
- 持续3天

**特殊处理**：
- 如果在章节末尾触发，直接进入坏结局
- 如果在章节中间触发，3天后恢复，但心情上限-10

### 存款归零

**触发条件**：
```
存款 <= 0
```

**结局类型**：中结局（经济困难线）

**结局效果**：
- 触发「缺钱」事件
- 心情-10
- 无法进行需要花费的活动
- 持续到存款恢复

**特殊处理**：
- 不会直接触发坏结局
- 但会影响结局评分

## 结局评分计算

### 评分维度

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 心情 | 30% | 心情 / 100 |
| 技能 | 30% | (编程等级 + 运营等级) / 10 |
| 存款 | 20% | 存款 / 10000 |
| 社交 | 20% | 社交能力 / 10 |

### 评分公式

```
结局评分 = 心情权重×0.3 + 技能权重×0.3 + 存款权重×0.2 + 社交权重×0.2

其中：
心情权重 = min(心情 / 100, 1.0)
技能权重 = min((编程等级 + 运营等级) / 10, 1.0)
存款权重 = min(存款 / 10000, 1.0)
社交权重 = min(社交能力 / 10, 1.0)
```

### 评分示例

**示例1：S级结局**
```
心情 = 80, 编程等级 = 3, 运营等级 = 0, 存款 = 5000, 社交能力 = 5

心情权重 = 80/100 = 0.8
技能权重 = (3+0)/10 = 0.3
存款权重 = 5000/10000 = 0.5
社交权重 = 5/10 = 0.5

结局评分 = 0.8×0.3 + 0.3×0.3 + 0.5×0.2 + 0.5×0.2
         = 0.24 + 0.09 + 0.10 + 0.10
         = 0.53
```

**修正后**：由于心情>60，触发好结局条件，直接进入A级结局

**示例2：C级结局**
```
心情 = 30, 编程等级 = 0, 运营等级 = 0, 存款 = 1000, 社交能力 = 2

心情权重 = 30/100 = 0.3
技能权重 = (0+0)/10 = 0.0
存款权重 = 1000/10000 = 0.1
社交权重 = 2/10 = 0.2

结局评分 = 0.3×0.3 + 0.0×0.3 + 0.1×0.2 + 0.2×0.2
         = 0.09 + 0.00 + 0.02 + 0.04
         = 0.15
```

**修正后**：由于心情<40，触发坏结局条件，直接进入D级结局

## 结局等级划分

### 等级定义

| 等级 | 评分范围 | 说明 | 奖励 |
|------|----------|------|------|
| S级 | 0.8-1.0 | 完美结局 | 特殊奖励 + 隐藏内容 |
| A级 | 0.6-0.8 | 优秀结局 | 标准奖励 + 后续优势 |
| B级 | 0.4-0.6 | 良好结局 | 基础奖励 |
| C级 | 0.2-0.4 | 普通结局 | 无奖励 |
| D级 | 0.0-0.2 | 悲剧结局 | 需要重玩 |

### 等级修正

**正面修正**（+0.1）：
- 心情 > 80
- 技能等级 >= 3
- 存款 > 8000
- 社交能力 >= 8

**负面修正**（-0.1）：
- 心情 < 30
- 技能等级 = 0
- 存款 < 2000
- 社交能力 < 3

**修正后评分**：
```
最终评分 = 基础评分 + 正面修正 - 负面修正
最终评分 = max(0, min(1, 最终评分))
```

## 结局与后续关联

### 结局对下一章的影响

| 结局等级 | 下一章起点 | 难度调整 |
|----------|------------|----------|
| S级 | 高起点 | 简单模式 |
| A级 | 标准起点 | 普通模式 |
| B级 | 低起点 | 困难模式 |
| C级 | 很低起点 | 噩梦模式 |
| D级 | 重玩 | 无影响 |

### 结局奖励

**S级奖励**：
- 下一章初始心情+20
- 下一章初始技能等级+1
- 解锁隐藏剧情
- 获得特殊成就

**A级奖励**：
- 下一章初始心情+10
- 下一章初始技能等级+1
- 解锁标准剧情

**B级奖励**：
- 下一章初始心情+5
- 解锁基础剧情

**C级奖励**：
- 无奖励

**D级奖励**：
- 解锁「轮回者」成就
- 重玩时获得提示

## 结局触发检测

### 检测时机

1. **章节结束时**：自动触发结局判定
2. **特殊事件时**：如健康归零、心情崩溃
3. **玩家选择时**：如主动结束章节

### 检测顺序

1. **检查强制结局**：健康、心情、存款归零
2. **检查主条件**：根据选择判断结局类型
3. **检查附加条件**：确定结局质量等级
4. **计算结局评分**：综合各项指标
5. **应用修正**：正面/负面修正
6. **确定最终结局**：根据评分和条件

### 结局冲突处理

**多结局冲突**：
- 如果同时满足多个结局条件，优先级：好结局 > 中结局 > 坏结局
- 同等级结局，评分高者优先

**强制结局优先**：
- 强制结局（健康归零等）优先级最高
- 会覆盖其他所有结局条件

## 调参建议

### 评分权重调整

- 心情权重不宜太高，避免玩家只关注心情
- 技能权重应较高，体现游戏核心玩法
- 存款权重适中，体现经济压力
- 社交权重可调整，影响人脉系统重要性

### 等级划分调整

- S级条件不宜太苛刻，让玩家有成就感
- A级应作为标准体验，让大部分玩家能达到
- B级应作为及格线，让玩家不失望
- C/D级应作为教训，让玩家想重玩

### 结局奖励调整

- 好结局奖励要明显，激励玩家追求完美
- 坏结局惩罚要适中，让玩家想重玩但不沮丧
- 隐藏结局奖励要特殊，增加探索乐趣

### 后续影响调整

- 结局对下一章的影响要明显，体现选择意义
- 影响要可持续，而不是一次性奖励
- 要给玩家改变命运的机会，而不是锁定结局
